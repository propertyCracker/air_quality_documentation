
# Warehouse / Analytics Layer

Purpose: Optimized storage for querying and reporting.

Why: Large-scale analytics require performance-tuned storage with indexing and partitioning.

Examples:

Aggregated device statistics for dashboards.

Summarized simulator results.

Format: Dedicated SQL pools (Synapse SQL) or serverless SQL over Parquet/Delta.

Features:

Columnar storage for fast analytics.

Partitioning by device, region, or time for efficient queries.

Indexing for frequently queried metrics.
